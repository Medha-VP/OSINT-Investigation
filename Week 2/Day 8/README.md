# Google Hacking Database (GHDB) Analysis

## Overview

This repository contains an analysis of the **Google Hacking Database (GHDB)** conducted as part of an Open-Source Intelligence (OSINT) learning program. The project explores different categories of Google Dorks and demonstrates how advanced search queries can be used to locate publicly available information while following ethical and responsible OSINT practices.

## Objective

- Understand the Google Hacking Database (GHDB).
- Explore different categories of Google Dorks.
- Learn how Google Dorks help discover publicly available information.
- Analyze the security implications of exposed online resources.
- Practice ethical and passive OSINT investigation techniques.

## Google Dork Categories Analyzed

The investigation explores Google Dorks across multiple categories, including:

- File Discovery
- Login Portals
- Sensitive Information
- Vulnerable Servers
- Configuration Files

Each category demonstrates how publicly indexed resources can be identified and the potential security risks associated with unintended information exposure.

## Investigation Methodology

The investigation was performed using publicly available Google search results and selected GHDB queries. The analysis focused on identifying:

- Public documents
- Login interfaces
- Directory listings
- Configuration files
- Log files
- XML and text files

No attempts were made to access restricted systems, authenticate to services, or exploit vulnerabilities.

## Key Findings

- Organizations often publish public documents for customers and employees.
- Login pages are easily discoverable through search engines.
- Open directory listings can expose unnecessary files.
- Public configuration and log files may reveal technical information.
- Proper website configuration is essential to minimize accidental information exposure.
